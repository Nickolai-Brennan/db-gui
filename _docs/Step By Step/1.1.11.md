## Step 1.1.11 — ERD Annotation Overlay (UI Behavior Spec) (exact)

Goal: make ERD **self-explanatory** by visualizing checklist results
directly on tables and relationships.

---

# 1.1.11a) Visual elements

Tables:
- Badge with count
- Badge color = max severity

Relationships:
- Small dot at midpoint
- Dot color = max severity

---

# 1.1.11b) Hover behavior

Hover table:
- Tooltip with top 3 issues
- “View all” link opens Inspector → Validation tab

Hover relationship:
- Tooltip with relationship-specific issues
- “Fix” button if fixAction exists

---

# 1.1.11c) Click behavior

Click table badge:
- Select table
- Open Inspector → Validation tab
- Filter issues to table scope

Click relationship dot:
- Select relationship
- Open Relationship editor → Validation tab

---

# 1.1.11d) Spotlight mode

Triggered by deepLink:
- Zoom to target
- Dim all unrelated nodes/edges to 20% opacity
- Highlight path for relationship issues

Exit spotlight on:
- ESC
- clicking empty canvas

---

# Done criteria for 1.1.11

✅ Badges and dots match annotation API  
✅ Hover tooltips accurate and fast  
✅ Clicks open correct inspector context  
✅ Spotlight mode guides user visually
